# Универсальный бот-информатор для инвестирования

## Предпосылки
При активном использовании стратегии [скальпинга](https://skillbox.ru/media/money/skalping-chto-o-nem-nuzhno-znat-i-kakoy-dohod-on-mozhet-prinesti/) я столкнулся с проблемой: постоянный мониторинг биржи отнимает много времени и внимания. Стандартное брокерское приложение уведомляет о достижении ценой заданного уровня, но мне не хватало двух ключевых функций:
1.  **Аналитики в разрезе отдельных сделок**, а не усреднённых показателей по активу.
2.  **Прогноза потенциального дохода** до совершения сделки, чтобы оценить её целесообразность.

Так появилась идея этого бота-информатора.

## Архитектура проекта
Проект построен на принципах **гексагональной архитектуры (Ports & Adapters)**. Бизнес-логика отделена от внешних источников данных, что позволяет легко адаптировать систему под любой API брокера или инвестиционной платформы.

```
/src
├── core/
│   └── ports.py                         # Абстрактный интерфейс (контракт) для провайдера данных
├── adapters/
│   ├── MoexPriceProvider.py             # Реализация контракта на основе открытого API Московской биржи
│   └── TelegramNotifier.py              # Адаптер для форматирования и отправки уведомлений
└── services/
    └── PositionMonitor.py               # Бизнес-логика: расчёт доходности, учёт количества акций и цены покупки
```

**Ключевые компоненты:**
*   `core/ports.py` — Определяет контракт, который должны выполнять все провайдеры данных.
*   `adapters/MoexPriceProvider.py` — Актуальная реализация с использованием бесплатного API MOEX.
*   `services/PositionMonitor.py` — Сервис-калькулятор, который, используя переданный провайдер, рассчитывает P&L и проверяет условия для срабатывания сигналов.

## Требования к окружению
*   Python 3.13 или выше

## Быстрый запуск (Quick Start)
1.  Клонируйте репозиторий: `git clone <URL>`
2.  Создайте виртуальное окружение: `python -m venv venv`
3.  Активируйте его.
4.  Установите зависимости: `pip install -r requirements.txt`
5.  Запустите приложение: `python main.py`

1.  **Интеграция с Telegram:** Превращение модуля в полноценного Telegram-бота.
2.  **Фоновая работа:** Запуск проверок и уведомлений по расписанию (интервалу в N минут).
3.  **Хранилище данных:** Интеграция с `aiosqlite` или аналогами для хранения данных о пользователях и их портфелях (с фокусом на быстрый запуск и конфигурацию).
4.  **Визуализация:** Разработка фронтенд-дашборда.
5.  **ИИ-советник:** Исследование возможности подключения локальной LLM (Qwen, аналогов) для анализа портфеля.
